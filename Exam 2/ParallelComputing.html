Parallel Computing

Basic Idea:

  • Take computations that can be done independently (or close to it)
  • Don't run sequentially
  • Split up computation
  • Run computation simultaneously on
    • different processor cores
    • across many connected computers (i.e on a cluster)
    • or a few other ways
  • Combine results

Parallel Computing

Need to predict on observations outside of our sample (training data)

  • If lots of data, split into training and test

  • Fit model on training data, evaluate on test set, compare to other methods to choose best model (retrain final model on full data)

  • If you lack data, can use Cross Validation

Aside: Cross Validation

k fold Cross Validation (CV)

  • Split data into k folds
  • Train model on first k-1 folds, test on kth
  • Train model on first k-2 folds and kth fold, test on (k-1)st fold
  • …
  • CV error = sum all test errors from above (overall misclassification)
  • Key = no tests by a model were done on data used to train that model!

library(tree)
splitData <- function(data, foldnum) {
  train <- filter(data, fold != foldnum)
  test <- filter(data, fold == foldnum)
  list(train = train, test = test)
}
fitTitanicTree <- function(train, test){
  fit <- tree(survived ~ pclass + sex + age + ticket + fare+ cabin + embarked, data = train)
  preds <- predict(fit, newdata = test, type = "class")
  misclass <- sum(abs(as.numeric(as.character(preds)) - as.numeric(as.character(test$survived))))
  return(list(fit = fit, preds = preds, misclass = misclass))
}

  • How to parallelize this in R?

  • Create function using lapply() to do the fitting/testing

getResult <- function(x, data){
  split <- splitData(data, foldnum = x)
  fitTitanicTree(split$train, split$test)  
}
results <- lapply(1:k, FUN = getResult, data = titanicData)
